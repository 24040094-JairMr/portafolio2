Tema 1

Geolocalización y sus Aplicaciones

Tecnologías y servicios para identificar la ubicación geográfica de dispositivos y usuarios

¿Qué es la Geolocalización?

Geolocalización es la identificación de la ubicación geográfica real de un objeto, como un teléfono móvil, computadora o dispositivo conectado a Internet. Se basa en el análisis de datos de posición obtenidos a través de diferentes tecnologías.

Tecnologías de Geolocalización

Existen múltiples tecnologías para determinar la ubicación de un dispositivo, cada una con diferentes niveles de precisión y aplicaciones específicas.

Tecnologías GPS

GPS (Global Positioning System)

Sistema de navegación por satélite que proporciona ubicación precisa usando 24+ satélites.

Precisión: 5-10 metros Cobertura: Global

Wi-Fi Positioning

Usa señales de redes Wi-Fi cercanas para triangulación de posición en interiores.

Precisión: 20-50 metros Uso: Interiores/urbanos

Triangulación de Celular

Calcula posición basándose en la distancia a torres de telefonía celular cercanas.

Precisión: 100-1000m Cobertura: Zonas pobladas

IP Geolocation

Estima ubicación basada en la dirección IP del dispositivo, usando bases de datos.

Precisión: Ciudad/región Uso: Aproximación

Aplicaciones que Utilizan Geolocalización

Navegación y Mapas

Google Maps, Waze, Apple Maps - Guían a usuarios de un punto a otro.

  • Rutas óptimas
  • Tráfico en tiempo real
  • Información de lugares cercanos

Comercio Electrónico

Amazon, Mercado Libre - Localizan usuarios para ofertas regionales.

  • Ofertas locales
  • Tiempos de entrega
  • Tiendas cercanas

Redes Sociales

Facebook, Instagram - Ubican publicaciones y encuentros sociales.

  • Etiquetas de ubicación
  • Eventos cercanos
  • Amigos en zona

Viajes y Turismo

Airbnb, Booking - Localizan alojamientos y atracciones turísticas.

  • Hoteles cercanos
  • Atracciones locales
  • Reservas geográficas

Casos de Uso Avanzados:

Servicios de Emergencia

Localización precisa para ambulancias, policía y bomberos.

Logística y Transporte

Seguimiento de flotas y optimización de rutas.

Juegos

Realidad aumentada y juegos basados en ubicación como Pokémon GO.

Análisis de Datos

Estudios demográficos y patrones de movilidad.

Seguridad y Privacidad en Geolocalización

Riesgos Comunes

  • Rastreo no autorizado de ubicación
  • Exposición de patrones de movilidad
  • Robo de identidad basado en ubicación
  • Acoso geográfico

Mejores Prácticas

  • Solicitar permiso explícito del usuario
  • Proporcionar control granular de permisos
  • Anonimizar datos cuando sea posible
  • Cifrar datos de ubicación

Importante: Las aplicaciones deben cumplir con regulaciones como GDPR y CCPA cuando manejan datos de ubicación de usuarios.

OAuth: Autenticación en APIs Sociales

Flujo OAuth 2.0:

1
Inicio de Sesión Social

Usuario elige iniciar sesión con red social

2
Autorización

Red social solicita permisos al usuario

3
Código de Autorización

Se redirige al usuario con código de autorización

4
Token de Acceso

Aplicación intercambia código por token de acceso

Acceso a API

Beneficios de OAuth:

Tema 3

Bases de Datos para Aplicaciones Web

Sistemas de almacenamiento y gestión de datos para aplicaciones modernas

Tipos de Bases de Datos

Bases de Datos SQL

Relacionales, estructuradas

  • MySQL, PostgreSQL, SQL Server
  • Estructura en tablas con relaciones
  • ACID compliance
  • Escalado vertical
Ideal para: Sistemas transaccionales, datos estructurados

Bases de Datos NoSQL

No relacionales, flexibles

  • MongoDB, Cassandra, Redis
  • Documentos, clave-valor, grafos
  • Alta escalabilidad horizontal
  • Flexibilidad en esquemas
Ideal para: Big Data, aplicaciones web modernas

Consideraciones al Elegir Base de Datos:

Volumen de Datos

¿Cuántos datos se manejarán?

Rendimiento

¿Qué velocidad de lectura/escritura se necesita?

Escalabilidad

¿Cómo crecerá el sistema?

Consistencia

¿Qué nivel de consistencia se requiere?

Bases de Datos como Servicio (DBaaS)

DBaaS (Database as a Service) son bases de datos gestionadas por proveedores cloud, eliminando la necesidad de administrar infraestructura.

AWS Database Services

  • Amazon RDS: MySQL, PostgreSQL, SQL Server
  • Amazon DynamoDB: NoSQL key-value
  • Amazon Aurora: Compatible con MySQL/PostgreSQL

Google Cloud Databases

  • Cloud SQL: MySQL, PostgreSQL, SQL Server
  • Cloud Firestore: NoSQL document database
  • Cloud Spanner: Base de datos distribuida

Azure Database Services

  • Azure SQL Database: SQL Server como servicio
  • Cosmos DB: Base de datos multimodelo
  • Azure Database for MySQL: MySQL gestionado

Tema 5

APIs de Geolocalización y Mapas

Implementación de mapas y servicios de ubicación en aplicaciones web

Principales APIs de Mapas

Google Maps Platform

Solución completa
  • APIs: Maps JavaScript, Places, Geocoding, Directions
  • Ventajas: Completa, precisa, amplia documentación
  • Precio: Gratis hasta cierto límite, luego pago por uso
  • Uso ideal: Aplicaciones empresariales, alta precisión

Leaflet.js

Biblioteca open source
  • Característica: Ligera, extensible, open source
  • Ventajas: Gratuita, plugins abundantes, personalizable
  • Precio: Completamente gratuita
  • Uso ideal: Proyectos pequeños/medianos, presupuesto limitado

Mapbox

Plataforma de mapas personalizables
  • Característica: Altamente personalizable, diseño moderno
  • Ventajas: Diseños atractivos, SDKs móviles, análisis
  • Precio: Gratis inicial, luego por uso
  • Uso ideal: Aplicaciones con marca fuerte, diseño personalizado

Generación y Gestión de Claves API

Proceso de Obtención de Clave API:

01

02
Crear Proyecto

Habilitar API

Activar las APIs específicas que se necesitan (Maps, Places, etc.)

04
Generar Credenciales

Crear una clave API o credenciales OAuth según sea necesario

05

Establecer restricciones por IP, referer HTTP, o aplicación

Buenas Prácticas de Seguridad:

Nunca exponer claves API en código frontend público
Usar restricciones por dominio/IP
Rotar claves periódicamente
Monitorear uso para detectar anomalías

// Inicializar mapa
function initMap() {
    // Ubicación inicial (Ciudad de México)
    const initialLocation = { lat: 19.432608, lng: -99.133209 };
    
    // Crear mapa
    const map = new google.maps.Map(document.getElementById('map'), {
        center: initialLocation,
        zoom: 12,
        mapTypeId: 'roadmap'
    });
    
    // Agregar marcador
    const marker = new google.maps.Marker({
        position: initialLocation,
        map: map,
        title: 'Ciudad de México'
    });
    
    // Búsqueda de lugares
    const searchBox = new google.maps.places.SearchBox(
        document.getElementById('search-box')
    );
    
    map.addListener('bounds_changed', () => {
        searchBox.setBounds(map.getBounds());
    });
    
    // Obtener ubicación actual
    document.getElementById('locate-me').addEventListener('click', () => {
        if (navigator.geolocation) {
            navigator.geolocation.getCurrentPosition(
                (position) => {
                    const pos = {
                        lat: position.coords.latitude,
                        lng: position.coords.longitude
                    };
                    map.setCenter(pos);
                    new google.maps.Marker({
                        position: pos,
                        map: map,
                        title: 'Tu ubicación'
                    });
                }
            );
        }
    });
}

// Inicializar cuando se cargue la página
window.onload = initMap;
